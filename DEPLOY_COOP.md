# Свой сервер для вкладки «Collab» — за 5 минут, без командной строки

Вкладка «Collab» показывает вам и вашим соавторам темы, вопросы и ответы
общего .siq-пака в реальном времени. Чтобы программы соавторов «видели» друг
друга, им нужна **одна общая точка обмена — маленький бесплатный сервер на
Cloudflare**. Поднимает его **один человек из группы**, а потом просто раздаёт
остальным адрес. Всё делается мышкой на сайте Cloudflare.

> Нужен всего один сервер на всю вашу компанию. Хостить его для чужих людей не
> надо — у каждой отдельной группы соавторов свой.

---

## Шаг 1. Аккаунт Cloudflare (бесплатно)

1. Зайдите на https://dash.cloudflare.com/sign-up и заведите аккаунт
   (почта + пароль, карта не нужна).
2. Подтвердите почту и войдите в панель.

## Шаг 2. Создать хранилище (KV)

Здесь сервер будет хранить обзоры паков.

1. В левом меню: **Storage & Databases → KV** (в старых версиях — просто «KV»).
2. Кнопка **Create a namespace** (Создать).
3. Имя — любое, например `coop`. Нажмите **Add / Create**.

Готово, хранилище создано. К нему мы вернёмся в шаге 4.

## Шаг 3. Создать сам сервер (Worker)

1. В левом меню: **Workers & Pages → Create** (Создать) → вкладка **Worker**
   (Create Worker).
2. Придумайте имя, например `si-hyx-coop` — оно попадёт в адрес. Нажмите
   **Deploy** (Развернуть) — Cloudflare создаст «пустой» рабочий сервер.
3. Нажмите **Edit code** (Изменить код / `< >`).
4. Слева удалите весь текст, который там есть, и **вставьте целиком** содержимое
   файла [`coop_worker.js`](coop_worker.js) (лежит рядом с программой / в
   репозитории).
5. Нажмите **Deploy** (Развернуть) вверху справа.

## Шаг 4. Привязать хранилище к серверу

Чтобы сервер видел KV из шага 2, их нужно связать. **Имя связки должно быть
ровно `COOP_KV`** — сервер обращается именно к нему.

1. Откройте свой Worker → вкладка **Settings** (Настройки) →
   **Bindings** (или «Variables & Bindings» / «Variables»).
2. **Add binding → KV namespace**.
3. Заполните:
   - **Variable name** (имя переменной): `COOP_KV` — строго так.
   - **KV namespace**: выберите то, что создали в шаге 2 (`coop`).
4. Сохраните (**Save / Deploy**).

## Шаг 5. Взять адрес и раздать соавторам

1. На странице Worker вверху есть его адрес вида
   `https://si-hyx-coop.ВАШ-ПОДДОМЕН.workers.dev` — скопируйте его.
2. В программе: **Настройки → Дополнительные вкладки → включить «Collab»**.
3. На вкладке в поле **«Сервер»** вставьте этот адрес.
4. Отправьте соавторам **тот же адрес** — они вставляют его у себя в это же поле.

---

## Как пользоваться (каждый соавтор)

1. **«Мой .siq…»** — укажите свою рабочую копию пака.
2. **Сервер** — общий адрес из шага 5.
3. **Комната** — любой общий код пака, одинаковый у всех (например
   `solevaya-volevaya`). Разные паки = разные коды.
4. **Имя** — ваш ник (у каждого свой, чтобы было видно, кто что сделал).
5. **Подключиться**.

Дальше всё автоматически: как только кто-то сохраняет свой .siq, остальные
через несколько секунд видят его темы/вопросы/ответы. Публиковать вручную не
надо. Опрос идёт, только пока вкладка открыта на экране, — сервер зря не
нагружается.

Подсветка:
- 🟢 зелёный — есть только у меня;
- 🔵 синий — есть только у другого;
- 🟡 жёлтый — тему делает несколько человек (возможный дубль);
- 🔴 красный — совпал ответ (точный дубль, кто-то уже сделал такой вопрос).

Медиа сам не передаётся — только пометка «В: фото / О: видео», чтобы понимать,
что в вопросе/ответе есть картинка, звук или ролик.

## Мелочи

- **Бесплатного тарифа хватает** маленькой группе с запасом (лимиты Cloudflare —
  100 000 чтений и 1000 записей KV в день; программа читает раз в 6 секунд и
  только когда вкладка открыта, а пишет лишь при сохранении пака).
- **Приватность.** Кто знает адрес сервера и код комнаты — видит её содержимое.
  Не публикуйте адрес где попало; для другого пака берите другой код комнаты.
- **Через 14 дней** без активности комната сама удаляется на сервере (TTL).
- Предпочитаете командную строку? В шапке [`coop_worker.js`](coop_worker.js)
  есть вариант деплоя через `wrangler`.
