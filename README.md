# SI-HYX

Универсальный медиа-комбайн под Windows: скачивание видео/аудио (YouTube, TikTok, Instagram и др. через yt-dlp), перекодирование видео в AV1, обработка звука, конвертация изображений (AVIF/WebP/PNG/JPG/ICO), монтаж (обрезка видео/аудио по волне с предпросмотром, кадрирование и удаление объектов с видео нейросетью), редактор фото (кисть, фигуры, текст, кадрирование, поворот/отражение, удаление объектов нейросетью LaMa и удаление фона нейросетью RMBG-2.0), объединение фото, Base64 и менеджер промптов. Плюс экспериментальные вкладки SiQuesterHYX и ShikimoriHYX. Есть браузерное расширение для отправки ссылок прямо в программу.

Текущая версия — **0.5.2**.

## Скачать

Готовую программу бери во вкладке **[Releases](../../releases)** → скачай архив с суффиксом **`-full`** (например `SI-HYX-v0.5.2-full.zip`), распакуй и запусти `SI-HYX.exe`.
Устанавливать ничего не нужно — ffmpeg, yt-dlp и deno лежат в папке `bin/`, а нейросетевые модели — в папке `models/` рядом с программой.

> В релизе три файла: `…-full.zip` (код + bin + models, ставится с нуля), `…-update.zip` (только код, для дельта-обновления) и `manifest.json`. Для первой установки нужен именно **full**; `update`/`manifest` программа использует сама при автообновлении.

## Возможности

Вкладки можно перетаскивать и сортировать; порядок запоминается. Часть вкладок по умолчанию выключена и включается в настройках.

- **Обработка** — сжатие и перекодирование медиа: видео в AV1 (SVT-AV1) с настройкой скорости, разрешения и FPS; аудиоэффекты (нормализация громкости/loudnorm, fade); конвертация изображений в AVIF/WebP/PNG/JPG/ICO с лимитом разрешения и размера файла (для прозрачных PNG корректно сохраняется альфа в AVIF). Очередь файлов, двойной клик — кодировать файл отдельно.
- **Загрузчик** — видео/аудио по ссылке через yt-dlp: выбор качества, обрезка по таймингам, субтитры, куки, режим «только аудио» (MP3/M4A). Поддержка проблемных площадок (TikTok) через nightly-канал yt-dlp и повторные попытки.
- **Монтаж** *(бета)* — обрезка видео/аудио по звуковой волне с зумом: точки IN/OUT (мышь, таймкоды, кадры), предпросмотр с плеером, покадровая перемотка со скраб-звуком, субтитры (внешние и встроенные, рендер в кадр как в VLC), полноэкранный режим. Быстрый режим (copy без перекодирования), Smart Cut (склейка по ключевым кадрам) и полный реэнкод (CPU/GPU). **Кадрирование видео** прямо на кадре, **удаление объектов/водяных знаков с видео** покадрово той же нейросетью LaMa, что и в фото, **пикселизация-проявление** (для угадаек), загрузка картинки как видео. Экспорт только звука в MP3, сохранение текущего кадра в PNG, смена озвучки. Исходные файлы можно удалять из Проводника прямо во время работы — программа не держит их намертво. Горячие клавиши: Space, I/O, ←/→, Ctrl+Z/Ctrl+Y.
- **Редактирование фото** — два режима:
  - *Редактирование фото* — кисть и ластик, кадрирование прямо на холсте, **поворот и отражение** (значки сверху холста), **фигуры как в Paint** (прямоугольник, эллипс, линия, стрелка — с заливкой или контуром), **текст системными шрифтами** (выбор шрифта и размера), выбор цвета (окно палитры, Alt-пипетка по картинке и пипетка по всему экрану), зум/панорама (WASD/стрелки — на любой раскладке), отмена/возврат (Ctrl+Z/Ctrl+Y). Отдельно — **удаление объектов и водяных знаков нейросетью LaMa** (закрасил лишнее кистью — оно убирается и фон дорисовывается) и **удаление фона нейросетью RMBG-2.0** (объект отделяется от фона, прозрачность сохраняется в PNG). Считается на CPU или GPU (CUDA/DirectML через onnxruntime). Сохранение без потерь.
  - *Объединить фото* — склейка нескольких изображений в одно.
- **Base64** — кодирование/декодирование файлов и текста, опция «Создавать .txt файл» на выходе.
- **Промпт** *(по умолчанию выключена)* — менеджер промптов: хранение и быстрый выбор заготовок.
- **SiQuesterHYX** *(экспериментальная, по умолчанию выключена)* — просмотр и работа с .siq-пакетами вопросов (с защитой от XXE/zip-slip при разборе файлов).
- **ShikimoriHYX** *(экспериментальная, по умолчанию выключена)* — поиск аниме/манги через API Shikimori с фильтрами (жанры, темы, статус) и превью.
- **⚙ Настройки** — Discord-канал, локальный сервер для браузерного расширения, обновление yt-dlp, автообновление программы, аппаратное декодирование видео, выгрузка нейросетевых моделей из ОЗУ при простое (вкл/выкл) и прочие опции.

## Браузерное расширение

В комплекте — расширение, которое отправляет ссылку с открытой страницы прямо в «Загрузчик». Для связи программа поднимает локальный HTTP-сервер (включается в настройках).

## Автообновление

Программа сама проверяет GitHub Releases. Если внешние ассеты (`bin/` — ffmpeg/yt-dlp/deno — и `models/` — нейросети) не менялись, качается только лёгкий `update`-архив (десятки МБ) вместо полного (сотни МБ); если менялись — полный. Старые версии и установки без манифеста всегда получают `full`-архив.

## Обновление yt-dlp

Если перестало качать с YouTube/TikTok — открой **⚙ Настройки → Обновить yt-dlp**. Обновится `bin/yt-dlp.exe` без переустановки программы.

## Сборка из исходников

Нужен Python 3.10+ и зависимости:

```bash
pip install -r requirements.txt
```

Запуск из исходников:

```bash
python main.py
```

Сборка в .exe (Windows):

```bash
build.bat
```

`build.bat` собирает строго по `SI-HYX.spec` (единый источник правды: иконка, hidden-imports, **исключения тяжёлых неиспользуемых пакетов** — torch/transformers/pandas/scipy и т.п., которые иначе раздували сборку на ~0.5 ГБ). Затем рядом с `.exe` кладутся внешние ассеты `bin/` и `models/`, считается их общий хеш (`bin/.binver`) и собираются три ассета релиза: `…-full.zip` (код + bin + models), `…-update.zip` (только код) и `manifest.json`.

> Папки `bin/` и `models/` намеренно **не** хранятся в репозитории из-за размера — их нужно положить рядом самому или взять из релиз-архива.
>
> `bin/` (ffmpeg, ffprobe, yt-dlp, deno):
> - ffmpeg/ffprobe: https://www.gyan.dev/ffmpeg/builds/ (ffmpeg-release-essentials)
> - yt-dlp: https://github.com/yt-dlp/yt-dlp/releases/latest/download/yt-dlp.exe
> - deno: https://github.com/denoland/deno/releases
>
> `models/` (нейросети ONNX для вкладки «Фото»): `lama_fp32.onnx` (удаление объектов, ~200 МБ) и `model_uint8.onnx` (удаление фона RMBG-2.0, ~360 МБ). Дополнительно нужны `numpy`, `opencv-python` и `onnxruntime` (либо `onnxruntime-gpu` / `onnxruntime-directml` — код сам подхватит GPU: CUDA → DirectML → CPU). Без моделей или без onnxruntime подвкладки показывают понятную заглушку, остальное работает как обычно.

## Структура кода

| Файл | Назначение |
|------|-----------|
| `main.py` | главное окно, точка входа, HTTP-сервер для расширения, автообновление |
| `config.py` | импорты, пути, константы, стили, версия |
| `utils.py` | утилиты (ffmpeg/ffprobe, cookies, deno) |
| `widgets.py` | кастомные виджеты, превью, лента файлов |
| `workers.py` | фоновые потоки: загрузка (yt-dlp) и обработка (ffmpeg) |
| `tabs.py` | вкладки: Обработка, Загрузчик, Base64, Промпт, Редактирование фото (кисть/фигуры/текст/кадрирование/поворот/удаление объектов/удаление фона) и объединение фото |
| `edit_tab.py` | вкладка «Монтаж»: волна, плеер, обрезка (copy/Smart Cut/реэнкод), кадрирование и удаление объектов с видео, прокси-предпросмотр |
| `lama_inpaint.py` | удаление объектов/водяных знаков нейросетью LaMa (ONNX) |
| `rmbg_bg.py` | удаление фона нейросетью RMBG-2.0 / BiRefNet (ONNX) |
| `libass_renderer.py` | рендер субтитров (libass) для монтажа |
| `shikimori_api.py`, `shikimori_tab.py` | вкладка ShikimoriHYX (поиск аниме/манги) |
| `siquester_tab.py`, `siquester/` | вкладка SiQuesterHYX (просмотр .siq) |
| `error_report.py` | отправка отчёта об ошибке |
| `taskbar.py` | прогресс в панели задач Windows |
| `make_manifest.py` | генерация `manifest.json` для дельта-обновлений |
